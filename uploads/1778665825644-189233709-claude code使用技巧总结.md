、# 💡 Claude Code 使用技巧总结

> [!tip] 核心原则
> - 配置好对应仓库的 claude.md 文件，每个项目都有自己的定制配置
> - 新的项目可以用“/init”自动生成
> - 按层级生成对应仓库的独立 api-skill，然后根据 api-skill 制作对应的 coding-skill
> - 随时让 AI 自己总结问题，把问题和注意事项主动更新到 skill 和 claude.md 完成自进化闭环

## 💬 核心交互

| 技巧         | 说明                     |
| ---------- | ---------------------- |
| `@文件名`     | 引用文件到上下文（避免手动粘贴）       |
| `/clear`   | 清理上下文，释放 token         |
| `/compact` | 压缩对话历史，保留重点            |
| `/fast`    | 切换快速模式（更快输出）           |
| `Tab` 补全   | 文件路径自动补全               |
| `/memory`  | 经常关注memory，定时更新和移除无用记忆 |
| `/resume`  | 恢复历史对话                 |

## ❓ 高效提问

```
❌ "帮我修复代码"
✓ "修复 m2m_can.c:L123 的 NULL 解引用，不要修改其他逻辑"
```

- **精确定位**：给出文件名 + 行号
- **限定范围**：明确说"只修改 X，不动 Y"
- **约束输出**：说"最小改动"、"不新增文件"

## ⚡ Slash 命令（Skills）

```bash
/commit          # 规范化 git 提交，任务结束后可以直接说把当前修改提交一下
/debug-target    # 推送到嵌入式设备调试，适用于支持adb的平台设备
/uts-coding      # UTS 项目编码助手
/bsl-api         # BSL 层 API 参考
/ask             # 向代码作者提问
/simplify        # 代码质量审查
```

## 🔀 并行工具调用

Claude 会自动并行执行无依赖的操作：
- 同时读多个文件（Glob + Read 并行）
- 搜索 + 读文件同时进行

**触发方式**：一句话列出多个需求即可

## 📋 Plan 模式

```
用户：帮我规划 XXX 功能开发
→ Claude 进入 Plan 模式
→ 探索代码库 → 生成 plans/xxx.md
→ 等待审批 → 执行
```

适合：复杂多文件修改、架构设计、不确定方案时

## 🧠 记忆管理

```
"记住：本项目一个编码任务结束的条件是编译通过"
"忘记之前关于 XX 的记录"
```

- 存储在 `~/.claude/projects/.../memory/MEMORY.md`
- 跨会话持久化，无需重复说明

## 🔐 权限模式

```bash
claude --dangerously-skip-permissions  # 全自动（CI 场景） root用户不让用。
```

- 默认模式：危险操作会弹出确认
- 敏感操作（git push、删除文件）始终建议确认

## 💾 上下文省 Token 技巧

1. **用 `/clear` 开新任务**，避免旧上下文干扰
2. **引用具体文件**而非粘贴大段代码
3. **Skill 调用**让子 Agent 处理重型搜索，保护主上下文
4. **说清楚限制**：减少来回修改的轮次
