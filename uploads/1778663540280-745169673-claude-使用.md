# Ubuntu 环境下配置 Claude Code 指南

本文档指导如何在 Ubuntu 环境下完成 Claude Code 的配置，确保 CLI 工具与 VS Code 插件协同工作，并正确加载项目自定义 Skill。

> 详细配置步骤请参考：[[Ubuntu下配置Claude]]
>

## 1. 安装 VS Code 插件

为了方便在终端提问的同时保留历史记录，建议首先安装官方插件。

> [!tip] 插件信息
> **插件名称**: `Claude Code for VS Code`
>
> **功能说明**: 在终端 CLI 中进行的对话记录会自动同步到该插件中，方便回溯和管理历史会话。

## 2. 部署项目配置 (claude_cfg)

将特定的 Claude 配置文件克隆到代码仓库的**根目录**下，这是 Claude Code 识别项目上下文和自定义技能的关键。

```bash
# 进入你的工作目录 (以 uts 为例)
cd uts

# 克隆配置仓库到当前根目录
git clone git@172.16.1.23:script/claude_cfg.git .

# 切换到 uts linux4.0 分支
git checkout -b develop/linux4.0_uts
```

> [!info]- 文件结构示意图
> ![[Pasted image 20260309110428.png]]

> [!important] 配置位置要求
> 请务必确保 `.claude` 文件夹或相关配置文件位于工程的**最顶层目录**，否则 Claude Code 将无法正确识别项目配置。

## 3. 启动与验证

启动 Claude 并确认自定义 Skill 是否生效。

### 启动命令

```bash
cd uts
claude
```

### 验证 Skill 加载

在 Claude 会话界面中输入 `/` 符号，检查是否弹出了 `uts_coding` 相关的技能简介。

> [!success]- 🔍 如何判断加载成功？
> 输入 `/` 后，若出现 **uts_coding skill** 提示，说明配置已正确读取。
>

## 4. 扩展：安装官方 Skills

Claude 官方提供了丰富的技能库，可根据项目需求选择性安装。

> [!info] 官方技能仓库
> **GitHub 地址**: [anthropics/skills](https://github.com/anthropics/skills)
>
> 包含文档处理、数据分析、开发工具等多种实用技能。

### 方法一：使用 Marketplace 自动安装 ⚠️

> [!warning] 不推荐
> 此方法会导入大量技能，每次加载都会消耗额外 token。

**步骤 1**: 添加官方技能市场

```bash
/plugin marketplace add anthropics/skills
```

**步骤 2**: 安装特定技能包

```bash
# 示例：安装文档处理技能集
/plugin install document-skills@anthropic-agent-skills
```

**优点**：
- ✅ 升级方便，使用 `/plugin` 命令即可更新

**缺点**：
- ❌ 导入技能过多，增加 token 消耗
- ❌ 无法精细控制所需技能

---

### 方法二：手动选择安装 ✅ 推荐

> [!tip] 推荐理由
> - 仅安装所需技能，节省 token 消耗
> - 放置在项目仓库中，Qwen 等其他模型也可使用
> - 完全掌控技能清单，便于团队协作

**操作步骤**：
- 进入uts工作区

```bash
# 1. 克隆官方技能仓库
git clone git@github.com:anthropics/skills.git

# 2. 复制所需技能到项目配置目录
cp -r skills/skills/docx .claude/skills/           # DOCX 文档读取
cp -r skills/skills/skill-creator/ .claude/skills/ # Skill 创建工具
cp -r skills/skills/xlsx/ .claude/skills/          # Excel 表格处理
cp -r skills/skills/pdf/ .claude/skills/           # PDF 文档读取

# 3. 清理临时仓库
rm -rf skills
```

> [!success] 安装完成
> 重新启动 Claude 后，输入 `/` 即可看到新增的技能
